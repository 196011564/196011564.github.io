    2. 2. 当你按下开机键时，你的电脑就会开始工作。在这一瞬间，主板会发送一个信号给电力支持装置。当它接受到了这个信号之后，电源（power supply device）便会提供电力给电脑。当主板接收到电源备妥信号（就是电源发出特定的电压）之后，它会尝试去启动CPU。CPU首先会重置他的寄存器，将预定设定的值放入寄存器中。
    3. 3. 在80386和其之后的CPU中，一般都定义了以下的CPU寄存器初始值
    4. 4. 当处理器开始实模式（real mode）时。首先让我们回顾一下这个模式下的内存段式管理（Memory Segmentation）。实模式是几乎兼容支持所有版本的x86处理器。范围覆盖到8086的32位处理器到英特尔的64位处理器。8086处理器有一个20位长度的寻址总线，这意味着他可以工作在0-0xFFFFF（1兆字节）空间内。但是它只有16比特的寄存器，这些寄存器最大值为2^16（0xFFFF）
    5. 5. 内存段式管理被应用到所有的地址空间，所有的内存被分割成65536字节（64KB）的small、fixed-size segments。因为我们不好用16比特寄存器去处理64KB的内存地址（因为16比特地址放不下64KB这么大的地址的值呀），所以它提供了另一种方案。
    6. 6. 一个地址由两个部分来确定：一个段选择器（它可以选择一个基地址），另一个则是基于这个基地址（Base Address）进行偏移。在实模式中，基地址和段选择器（Segment Selector）之间的关系是是Base Address = Segment Selector * 16。因此要计算出内存的物理地址，我们需要将段选择器乘16，然后加上偏移。

  • 引导程序
    1. 1. 在现实世界中，要启动 Linux 系统，有多种引导程序可以选择。比如 GRUB 2 和 syslinux。Linux内核通过 Boot protocol 来定义应该如何实现引导程序。在这里我们将只介绍 GRUB 2。
    2. 2. 现在 BIOS 已经选择了一个启动设备，并且将控制权转移给了启动扇区中的代码，在我们的例子中，启动扇区代码是 boot.img。因为这段代码只能占用一个扇区，因此非常简单，只做一些必要的初始化，然后就跳转到 GRUB 2’s core image 去执行。 Core image 的代码请参考 diskboot.img，一般来说 core image 在磁盘上存储在启动扇区之后到第一个可用分区之前。core image 的初始化代码会把整个 core image （包括 GRUB 2的内核代码和文件系统驱动） 引导到内存中。 引导完成之后，grub_main将被调用。
    3. 3. grub_main 初始化控制台，计算模块基地址，设置 root 设备，读取 grub 配置文件，加载模块。最后，将 GRUB 置于 normal 模式，在这个模式中，grub_normal_execute (from grub-core/normal/main.c) 将被调用以完成最后的准备工作，然后显示一个菜单列出所用可用的操作系统。当某个操作系统被选择之后，grub_menu_execute_entry 开始执行，它将调用 GRUB 的 boot 命令，来引导被选中的操作系统。
    4. 4. 就像 kernel boot protocol 所描述的，引导程序必须填充 kernel setup header （位于 kernel setup code 偏移 0x01f1 处） 的必要字段。kernel setup header的定义开始于 arch/x86/boot/header.S：
